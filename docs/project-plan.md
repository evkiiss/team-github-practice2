## Правила команды
1. Перед началом работы каждый участник должен сделать Pull.
2. Перед отправкой изменений каждый участник должен проверить статус Git.
3. Один Pull Request должен решать одну понятную задачу.
4. Названия коммитов должны быть короткими и понятными.
